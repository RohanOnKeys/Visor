Primary goal
Turn raw webpage DOM into safe, structured, agent-ready context.
MVP boundary
Chrome extension, local-only compiler, JSON/Markdown export, privacy redaction, tests.
1. Backend stance
MVP requires no backend. A backend is allowed only in later versions for opt-in sync, team workspaces, saved context
history, API access, or AI-assisted compilation. Raw page content must never be sent to a backend by default.
2. Future backend use cases
•
Encrypted user account sync for settings and site profiles.
•
Opt-in saved context history.
•
Team workspaces and shared extraction profiles.
•
Server-side heavy compilation for large pages only after consent.
•
API endpoints for tools that want AgentContext programmatically.
•
Billing and usage dashboard in a future commercial version.
