1. Testing philosophy
Testing must focus on compiler correctness, privacy safety, and browser-extension reliability. The MVP is accepted only
when representative fixture pages compile without crashes and generated AgentContext outputs validate against schema.

5. Example unit tests
describe('redactor', () => {
  it('masks email and JWT-like tokens in strict mode', () => {
    const input = 'Contact rohan@example.com token eyJhbGciOiJIUzI1NiJ9.x.y';
    const result = redact(input, { level: 'strict' });
    expect(result.text).not.toContain('rohan@example.com');
    expect(result.redactedItems).toContainEqual(expect.objectContaining({ type: 'email' }));
    expect(result.redactedItems).toContainEqual(expect.objectContaining({ type: 'jwt' }));
  });
});
6. Release blocking bugs
•
Any password value appears in output.
•
Any compile path makes an external network call in MVP.
•
AgentContext is invalid for normal fixture pages.
•
Extension requests broad host permissions without justification.
•
Preview UI renders extracted HTML as executable HTML.
•
Compile freezes browser on large fixture instead of degrading gracefully.
