Chrome extension, local-only compiler, JSON/Markdown export, privacy redaction, tests.
1. Storage principle
The MVP stores settings and site profiles locally. It should not store full page contents by default. Full compiled contexts
may exist temporarily in memory for preview and clipboard export.
2. Local storage objects
type UserSettings = {
  defaultMode: 'compact' | 'detailed' | 'agent_action' | 'rag' | 'debug';
  privacyLevel: 'low' | 'medium' | 'strict';
  tokenBudget: number;
  defaultExport: 'json' | 'markdown' | 'prompt_block';
  debugMode: boolean;
  blockedDomains: string[];
};
type SiteProfile = {
  id: string;
  domain: string;
  preserveSelectors: string[];
  ignoreSelectors: string[];
  mainContentSelector?: string;
  privacyLevelOverride?: 'low' | 'medium' | 'strict';
  createdAt: string;
  updatedAt: string;
};
type RecentCompileMetadata = {
  id: string;
  url: string;
  title: string;
  createdAt: string;
  mode: string;
  tokenCount: number;
  riskLevel: string;
};

6. Do-not-store list
•
Password field values.
•
OTP values.
•
Session tokens or cookies.
•
Full private dashboard content by default.
•
Banking, medical, government, or educational portal content without explicit save action.
•
Authentication headers or browser cookies.
