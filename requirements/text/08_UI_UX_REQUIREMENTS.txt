Primary goal
Turn raw webpage DOM into safe, structured, agent-ready context.
MVP boundary
Chrome extension, local-only compiler, JSON/Markdown export, privacy redaction, tests.
1. UX principle
The UI must feel like a compiler control panel, not a chatbot. The user should understand what page is being compiled,
what mode is selected, what privacy risk exists, and what output will be copied or exported.

5. Settings page
•
Default output mode: compact, detailed, agent_action, rag, debug.
•
Default privacy level: low, medium, strict.
•
Token budget: numeric preset or custom input.
•
Ignored selectors: user-defined CSS selectors.
•
Preserved selectors: user-defined CSS selectors.
•
Blocked domains: never compile list.
•
Debug mode toggle.
•
Clear local data button.
6. Interaction states
State
UI behavior
Idle
Show active page title and Compile button.
Compiling
Disable button, show progress text, allow cancel when possible.
Success
Show preview, token count, privacy status, copy buttons.
Partial success
Show warning banner and available partial context.
Restricted page
Explain that browser-internal pages cannot be compiled.
High privacy risk
Require deliberate export click; show redaction status.
7. Accessibility requirements
•
Keyboard accessible controls.
•
Visible focus states.
•
High contrast readable text.
•
ARIA labels for icon-only buttons.
•
No tiny-only color-coded risk indicators; include text label.
•
Copy/export result should announce success/failure textually.
